PayPal Website Payments Pro and Express Checkout modules for osCommerce 2.2 RC2A

Released 18th December 2009 (Build 177)

Updated 8th February 2010 (Build 180)

Build 180: Adds support for rp_ipn.php

These payment modules add support for additional features of PayPal Website Payments Pro and PayPal Express Checkout to osCommerce 2.2 RC2A, beyond the standard Website Payments Pro and Express Checkout modules included in osCommerce.

Including:

·         Direct Payments for debit and credit cards

·         PayPal Express Checkout (purchase without the buyer needing to pre-register on the osCommerce store - the customer account is created automatically)

·         Express Checkout Recurring Payments

·         Debit and credit card Recurring Payments

·         Fraud Management Filters

·         3D Secure Buyer Authentication via Cardinal Commerce (currently UK merchants only)

This release supports the English only version of osCommerce 2.2 RC2A.

WARNING: This module redistributes the un-customized default versions of certain files (as listed in this document) including cart, product information and checkout pages.  If you have customized the design of your osCommerce store, you will need to merge these design changes into the new module files, or if you prefer merge the code changes into your existing design.

NOTE: This module is designed for PayPal merchants using Website Payments Pro and/or Express Checkout with API Signatures.  It is not for use by merchants using Website Payments Pro (Payflow Edition) or Payflow Pro.  If you use https://manager.paypal.com to manage your Website Payment Pro account, please use the Website Payments Pro (Payflow Edition) or Payflow Pro module for osCommerce, rather than this module as it will not be compatible with your Website Payments Pro account.

Installation

To install the module, back up your existing osCommerce installation files and database.

Copy/upload all files in the /catalog/ directory over your existing osCommerce /catalog/ directory.  This will replace the updated the Website Payments Pro and Express Checkout files in your current osCommerce installation and add the new files to support additional features.

If you have customised the design or program code of your osCommerce installation, you will need to manually compare the updated files with your existing files, and manually merge any code or design changes.

If you are installing osCommerce for the first time, the installation instructions are available at http://www.oscommerce.info/kb/225, or http://www.oscommerce.info/kb/osCommerce/Installation_and_Upgrades/41 depending on your environment. You will need to turn on register_globals and enable the MySQL extension in php.ini, at a minimum.

Once the files have been copied/uploaded and/or merged into your existing osCommerce store design, you will need to Install (activate) the modules in the osCommerce Administration panel. To do this:

1.      Login to osCommerce administration for your store

2.      Select the Modules section

3.      Select 'Payment'

4.      If you are using Website Payments Pro, select PayPal Website Payments Pro, otherwise if you do not have a Website Payments Pro account, select PayPal Express Checkout

5.      Click Install in the Right column.

This will create the necessary database tables needed for the PayPal modules to operate, as well as activate the module within your osCommerce store.

Configuration Settings for the Website Payments Pro and Express Checkout modules

Enable {module name}

Enable or disable the payment module.

Website Payments Pro Direct Payments Only: Country Module

Select the country of your Website Payments Pro merchant account. This option will change the available credit and debit cards available to the buyer during checkout.

Enable PayPal Recurring Payments

Enable or disable Recurring Payments. Note: This will not disable any existing Recurring Payments already created.

Recurring Payments Autobill

Enable autobill to have PayPal attempt to bill outstanding amounts.

Recurring Payments Maximum Failed Payments

The maximum number of failed scheduled payments before the profile is automatically suspended.

Enable PayPal Fraud Management Filters Check

Enables the PayPal Fraud Management Filters.  You must also be signed up for Fraud Management Filters and have activated this on your Website Payments Pro account.

API Username

Used to execute transactions against your PayPal account.

Login to your PayPal account.

If you are using Premiere or Business account for PayPal Express Checkout:

Profile >  API Access > API Credentials > Request API Signature

If you are using PayPal Website Payments Pro:

Profile > Request API Credentials > Option 1  PayPal API > 2. Set up PayPal API credentials and permissions > API Credentials > Request API Signature

API Password

Available in your PayPal account, see API Username instructions above.

API Signature

Available in your PayPal account, see API Username instructions above.

 

Live or Sandbox API

Select Sandbox to run test transactions in conjunction with the PayPal test environment at https://developer.paypal.com.  If you intend to use Sandbox, you will need to login to https://developer.paypal.com, generate Buyer and Seller accounts at and configure the Seller accounts API Username, API Password and API Signature into osCommerce instead of using the API credentials from your live account.

When youre ready to process live transactions, ensure you select Live and update the API Username, API Password and API Signature from your live PayPal account.

Transaction Method

osCommerce can charge the customer immediately (this is a Sale transaction), or as an Authorization. If you use Authorization, you must Capture the authorization in a separate step before shipping to transfer the funds to your PayPal account.  The Authorization can be captured via the PayPal account.

Payment Zone

If a zone is selected, only enable the payment method for that zone.

Set Order Status

Sets the status of orders made with this payment module in the osCommerce order manager.

Express Checkout Module Only: Automatic Account Creation

If a buyer uses Express Checkout without logging in, and no osCommerce customer account exists for that email address, a new temporary customer account is created for the purposes of fulfilling this order. Enable this option if you would like the make the customer account permanent, and automatically email the login details to the buyer.

Sort order of display

The sort order for this payment method on the payment information page. Lowest is displayed first.

cURL Program Location

Set the location of the cURL application on your web server.

Log File Location

Set the location of the PayPal transaction log. Credit cards and CVV numbers will not be stored in the log.

NOTE: Ensure the server can write to this file location.

Website Payments Pro Direct Payments Only: Enable Cardinal Centinel Module (UK Selected Country Module only)

Enable 3D Secure Buyer Authentication via the Cardinal Centinel Payer module. You must have a Cardinal Centinel account to use this feature.

Website Payments Pro Direct Payments Only: Enable Debug Logging to Error Log (UK Selected Country Module only)

Enable logging for the Cardinal Centinel Module (stored in Log File Location).

Website Payments Pro Direct Payments Only: Transaction URL (UK Selected Country Module only)

Endpoint URL issued by Cardinal Commerce.

Website Payments Pro Direct Payments Only: Processor Id (UK Selected Country Module only)

Processor ID issued by Cardinal Commerce.

Website Payments Pro Direct Payments Only: Merchant Id (UK Selected Country Module only)

Merchant ID issued by Cardinal Commerce.

Website Payments Pro Direct Payments Only: Transaction Password (UK Selected Country Module only)

Transaction Password issued by Cardinal Commerce.

Website Payments Pro Direct Payments Only: Only Accept Chargeback Protected Orders (UK Selected Country Module only)

Select whether you'd like to proceed with the transaction when the Cardinal Centinel authentication result is unsuccessful.

Uninstalling

To remove the payment module from your osCommerce store:

1.      Login to osCommerce administration for your store

2.      Select the Modules section

3.      Select 'Payment'

4.      If you are using Website Payments Pro, select PayPal Website Payments Pro, otherwise if you do not have a Website Payments Pro account, select PayPal Express Checkout

5.      Click Remove in the Right column.

This will leave the previous data, and the administration features in place so you can still administer purchases used with the PayPal modules.
